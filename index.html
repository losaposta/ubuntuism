Goal of Ubuntuism

To be able to create one-of-a-kind Joomla! templates more quickly without imposing creative limitations.

Background

My personal experience when learning how to create Joomla! templates.

  • After I created my first one, I found myself reusing the same blocks of code
  • Start new templates by stripping unique parts from the last template
  • Eventually built a re-usable code base (a.k.a. skeleton template) in 2008
  • Construct born in January 2009, public in 2010, evolving since

Achieving Ubuntuism

Process & Mind-set

  • Start every template with same foundational code (i.e. framework)
  • Continuously identify code that can be reused and improved
  • Isolate unique aspects of every template per project basis
  • Kaizen - Japanese philosophy of continuous improvement wikipedia.org/wiki/Kaizen

Ubuntuism in Action

Approach

Use with existing framework or develop your own

  • Use or customize an existing framework
    • Morph - Joomlajunkie
    • Warp - YOOtheme
    • Gantry - RocketTheme
    • T3 - Joomlart
    • YjSimpleGrid - Youjoomla
    • Rumi - J!Labs
    • motif - Themeables
    • Wright - Joomlashack
    • Construct - Construct-Framework.com
    • ... etc

Approach

Use with existing framework or develop your own

  • Developing your own can happen through natural, evolutionary process of reviewing past work, reusing and improving it
  • Tips
    • Separate base css: resets/normalization, base typography, positioning, core styles, clearfix...etc
    • Centralize base logic, custom parameters (e.g. Google Web Font), specialized functionality
    • Provide comprehensive core layout
    • Add unique styles in separate (custom) style sheet => preserve customization with upgrade
    • Mechanism to override core layout => preserve customization with upgrade
    • Upgradability is critical for efficiently improving code and deploying it to existing sites

Let's code!
