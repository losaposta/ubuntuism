What is Ubuntuism?

The Process & Philosophy

  • Start every template with same foundational code (i.e. framework)
  • Continously identify code that can be reused and improved
  • Kaizen - continuous improvement philosophy wikipedia.org/wiki/Kaizen
  • Isolate unique aspects of every template per project basis

Background

  • Found myself reusing the same blocks of code.
  • Start new templates by stripping unique parts from the last template
  • Eventually built a base (a.k.a. skeleton template) in 2008
  • Construct born in January 2009, public in 2010, evolving since

Approach

Use with existing framework or develop your own

  • Seperate base css: resets/normalization, base typography, positioning, core styles, clearfix...etc
  • Centralize base logic, custom parameters (e.g. Google Web Font), specialized functionality
  • Provide comprehensive core layout

Approach

  • Add unique styles in seperate (custom) style sheet => preserve customization with upgrade
  • Mechanism to override core layout => preserve customization with upgrade
  • Upgradability is crtical for effeciently improving code
